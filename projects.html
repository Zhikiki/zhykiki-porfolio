myFlix full-stack project

Case Study

Introduction

myFlix is a web application developed with the MERN stack. It provides users with access to information about different movies, including information about the film’s director as well as which genre specific movie belongs to. Users can sign up, update their personal information, create a list of their favorite movies. Project’s back-end part also provide the possibility to implement the ability to add new movies to the movie library and update movie’s information.

Live Demo

Purpose

This project was built as part of my web development course at CareerFoundry to demonstrate mastery of full stack JavaScript development.

Objective

The aim was to create ambitious full-stuck project by building complete server-side and client-side from scratch to add to my professional portfolio.

Server Side

I created a RESTful API using Node.js and Express, that interacts with a non-relational database (MongoDB). The API can be accessed via commonly used HTTP methods like GET or POST. To retrieve and store data in the database, CRUD methods are used. The API provides movie information in JSON format.

Check on GitHub

  1. As far as I was already familiar with JavaScript, I choose Node runtime environment for back-end development process. I used Express framework to simplify Node.js syntax and to create and maintain web servers as well as manage HTTP requests

  2. After practicing with PostGreSQL relational database appeared too much restrictive. With idea of increasing flexibility to adapt to new requirements in the future, I choose NoSQL document database MongoDB and configured schema and models with Mongoose.

  3. To ensure only logged in users can access the database, I implemented a login endpoint  with basic HTTP authentication returning an authorization JWT (token) to the client. For extra user’s security I added CORS password hashing and data validation.

  4. At first, I felt like a blind developer, not being able to check the functionality of the created endpoints in the usual way. Using Postmen for testing the API has greatly simplified the task. I finally used Heroku to deploy my app and MongoDB Atlas to host my database.

Client Side

Once the backend was set up, for client side I developed single-page application and used state-based routing which allowed users to navigate to the different views. Application provides several interface views including a movie view, a login view, a registration view and a profile view (where users can update their user data and list of favorites).

Initial Build
The goal of this project was to understand how to use MVC architecture as a design pattern. To facilitate this, I chose to use React. It is fast, easy to maintain, and well documented. As far as initial build was done, I created components for different views. To build my app I choose the easiest option – Parcel. I recognize it as self-proclaimed super fast, no config web application bundler.
Components and State
As far as initial build was done, I created components for different views. I used Axios to pull in the API I had previously created. I also practiced with React hooks, which enabled function components to have access to state.
Styling
I used React Bootstrap to design the layout of the pages and cards and to ensure consistent and responsive styling. This library simplified a lot creation of some elements, like forms and cards.
State Management
I decided to centralize state and logic by using Redux. It highly increased level of complexity because I needed to find the way to refactor my code. But it all made a lot of sense cause the app became more scalable and behaves more consistently

Retrospective

  • What went well?

    I enjoyed a lot building server side of the app with strict logic, order and structure. I was also excited to get familiar with React most likely because it It makes it possible to work effectively with each individual component and element of the app, structures them, so that later they can be put together like a puzzle.

  • What didn't go well?

    In pursuit of the ideal, I always lose a lot of time on the design of the application, which shouldn't be overcomplicated. Issues were also caused by the necessity to refactor the code due to Redux implementation on the last stage of app development. Some features partially lost functionality and because of strict timing I haven’t fixed it in terms of course timeline.

  • What's next?

    I would like to polish app’s functionality. To provide natural growth of database I want to develop forms for adding and editing new movies. It would be better if user will be asked to enter their password before changing user data in Profile View. During registration I would like to implement the logic that will check if username is already taken.

  • Final thoughts

    In general, I am pretty proud of the achieved result. In the future, I would like to try myself with a few more projects using MERN stack. It will help me to get more familiar with used technologies and will help to develop confidence and ability to produce my own technical solutions.
